說明

此份文件目的是定義一套程式命名原則，使得程式碼具備一致性、可維護性。不同人員修改程式時更容易理解程式碼，使得焦點集中於開發新功能與快速找出問題錯誤，避免程式難以理解、花時間猜測程式內容、詢問前一維護者等問題。除此之外，文件中加入邏輯技巧，避免很多莫名其妙的 bug 產生、減少花時間去 debug。透過此項方法提高維護品質、降低維護上的困擾。

類和介面 (Classes and Interfaces)

類 (Classes)

使用帕斯卡命名法（PascalCase），每個單詞的首字母大寫。 (Use PascalCase, with the first letter of each word capitalized.)

範例 (Examples): Customer, OrderProcessor, DataManager

介面 (Interfaces)

使用帕斯卡命名法（PascalCase），並以大寫字母 I 開頭。 (Use PascalCase, prefixed with the letter I.)

範例 (Examples): IRepository, ILogger

列舉 (Enums)

使用帕斯卡命名法（PascalCase）。 (Use PascalCase.)

public enum Color {
    Red,
    Green,
    Blue
}

結構 (Structs)

使用帕斯卡命名法（PascalCase）。 (Use PascalCase.)

範例 (Examples): Point, Rectangle

方法 (Methods)

使用帕斯卡命名法（PascalCase）。通常是動詞或動詞片語。 (Use PascalCase. Typically verbs or verb phrases.)

範例 (Examples): CalculateTotal, SaveChanges, GetCustomerData

屬性 (Properties)

使用帕斯卡命名法（PascalCase）。通常是名詞或名詞片語。 (Use PascalCase. Typically nouns or noun phrases.)

範例 (Examples): FirstName, OrderDate, TotalAmount

修飾詞 (Modifiers)

私有 (Private)

使用駝峰命名法（camelCase），並以底線 _ 開頭。 (Use camelCase, prefixed with an underscore _.)

範例 (Examples): _totalAmount, _customerName

受保護 (Protected)

使用帕斯卡命名法（PascalCase）。 (Use PascalCase.)

範例 (Examples): ProtectedField

內部 (Internal)

使用帕斯卡命名法（PascalCase）。 (Use PascalCase.)

範例 (Examples): InternalField

受保護內部 (Protected Internal)

使用帕斯卡命名法（PascalCase）。 (Use PascalCase.)

範例 (Examples): ProtectedInternalField

私有受保護 (Private Protected)

使用帕斯卡命名法（PascalCase）。 (Use PascalCase.)

範例 (Examples): PrivateProtectedField

常數 (Constants)

使用帕斯卡命名法（PascalCase）。通常是名詞或名詞片語，並使用 const 關鍵字定義。 (Use PascalCase. Typically nouns or noun phrases, defined using the const keyword.)

範例 (Examples): MaxValue, DefaultTimeout

事件 (Events)

使用帕斯卡命名法（PascalCase）。通常是名詞或名詞片語，表示動作或狀態的變化。 (Use PascalCase. Typically nouns or noun phrases representing actions or state changes.)

範例 (Examples): DataReceived, PropertyChanged

區域變量和參數 (Local Variables and Parameters)

使用駝峰命名法（camelCase）。名稱應該簡潔且有意義。 (Use camelCase. Names should be concise and meaningful.)

範例 (Examples): orderCount, customerName, totalAmount

綜合範例 (Comprehensive Example)

public interface ILogger
{
    void Log(string message);
}

public enum LogLevel
{
    Debug,
    Info,
    Warn,
    Error,
    Fatal
}

public struct Point
{
    public int X { get; set; }
    public int Y { get; set; }
}

public class ExampleClass
{
    // Private field
    private string _privateField;

    // Protected field
    protected string _protectedField;

    // Internal field
    internal string InternalField;

    // Public property
    public string PublicProperty { get; set; }

    // Constant
    public const int MaxValue = 100;

    // Event
    public event EventHandler DataReceived;

    // Method
    public void PublicMethod() { }

    // Protected method
    protected void ProtectedMethod() { }

    // Internal method
    internal void InternalMethod() { }

    // Private method
    private void PrivateMethod() { }
}
